# SAM3 器具マスク生成 依頼書（Cholec80 / Surgformer 用）

このドキュメントだけで作業が完結します。手術器具（surgical instruments）の
セグメンテーションマスクを、以下の仕様で画像ファイルとして出力してください。

---

## 1. 出力してほしいもの
各フレーム画像に対する **器具 / 非器具のマスク**を、1 フレーム 1 枚の PNG で保存。
- クラス分け・インスタンス分離は **不要**。複数器具は **1 枚に OR 統合**。
- 値:
  - 2 値で出す場合 … 器具 = 255 / 背景 = 0
  - 尤度で出す場合 … 0..255（0 = 背景, 255 = 器具確信）の単一チャンネル

## 1.5. プロンプト（確定）
SAM3 のテキストプロンプトは **`metallic instrument`** を使用（相性確認済み）。
全器具を 1 枚のマスクに OR 統合する方針はこのプロンプト 1 本で満たす想定。

## 2. 入力画像（厳守）
分類器が実際に読み込む画像と**同一ファイル**にセグメンテーションをかけてください。

```
入力: {DATA}/frames_cutmargin/{video_id}/<filename>.png
例:   {DATA}/frames_cutmargin/video49/00042.png
```

- `{DATA}` は Cholec80 データのルート（Docker 内は `/workspace/data/Cholec80`）。
- `frames_cutmargin/` が存在しない video のみ `frames/{video_id}/` を入力に使う。
- **リサイズ・クロップ・パディングを一切かけない**。入力 PNG の解像度 (H×W) のまま処理。
  （224×224 化やパッチ化は分類器側で行う。SAM3 側は原寸のまま）

## 3. 出力先・命名（最重要）
**入力ファイル名をそのまま 1:1 でコピー**して、ディレクトリ名だけ差し替える。

```
出力: {DATA}/instrument_masks/{video_id}/<同じfilename>.png
例:   {DATA}/instrument_masks/video49/00042.png
```

- ファイル名は入力と**完全一致**（連番に振り直さない）。
  ※ 分類器側はパス文字列の `frames_cutmargin`(or `frames`) を `instrument_masks` に
    置換してマスクを探す。ファイル名が 1 文字でも違うと対応が取れない。
- 解像度も入力と**完全一致**（1:1 ピクセル対応）。
- 単一チャンネル（グレースケール）PNG。

## 4. 対象範囲（段階的に）
1. **まず video49 のみ**全フレーム出力 → 分類器側で整合チェック（パッチに正しく乗るか）。
2. OK なら評価対象 **video49〜80**（test split）へ拡大。
3. （学習にも使う場合は全 video。ただし現フェーズは推論時のみなので test split で十分）

## 4.5. 器具が写っていないフレーム
器具が検出されないフレームも **全黒 (全画素 0) のマスクを必ず出力**してください
（スキップしない）。入力フレームとマスクの枚数・ファイル名を 1:1 に保つため。

## 5. 整合チェック（出力前に SAM3 側で確認）
- 入力 PNG とマスク PNG を重ねて、器具の上にマスクが乗っているか目視。
- 入力とマスクで **解像度・ファイル名・枚数が一致**しているか。
- リサイズが入っていないか（最頻の事故ポイント）。

---

## 補足（背景・なぜこの仕様か）
- 分類器（Surgformer）は各フレームを 224×224 にリサイズし、16px パッチ（14×14=196）に
  分割して Spatial Attention を計算する。器具マスクはこのパッチ単位の被覆率に変換され、
  Attention の重みに加算されて「器具領域への注目」を強める。
- そのため必要なのは **原寸・原ファイル名のマスク画像のみ**。座標変換やパッチ化は
  すべて分類器側が担当するので、SAM3 側は通常のセグメンテーション出力を上記の場所に
  置くだけでよい。
